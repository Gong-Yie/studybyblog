好算法的标准

一个优质的算法需满足五大核心特性，是衡量算法设计优劣的基本准则：

  1. 正确性：能满足具体问题的需求，对合法输入输出正确结果，对非法输入能做出适配处理。

  2. 健壮性：具备对非法输入的抵抗能力，不会因错误输入产生错误动作或陷入瘫痪。

  3. 可读性：算法易于理解和实现，便于后续的调试、修改与维护。

  4. 时间效率高：算法运行的时间尽可能短，核心关注算法的执行速度。

  5. 空间效率高：算法运行过程中占用的计算机存储空间尽量少。

算法分析的基本概念

算法分析的定义与目的

  • 算法设计：面对具体问题，设计出能够解决问题的有效算法。

  • 算法分析：对已设计完成的算法，从多维度评价和判断其优劣，其结果可指导改进算法设计，也能检验评估算法的有效性。

  • 核心分析维度：效率（速度）是算法的核心和灵魂，同时兼顾正确性、健壮性、易读性、可扩展性等。

影响程序运行时间的因素

算法最终运行的时间并非单一因素决定，而是由多方面共同影响：

  1. 计算机系统性能：不同硬件的运算速度、存储能力存在差异（如天河三号百亿亿次 / 秒与个人电脑十亿次 / 秒的运算差距）。

  2. 程序所依赖的算法：算法的设计思路和执行逻辑是决定运行效率的核心因素。

  3. 问题规模和输入数据：问题的输入规模大小、输入数据的分布特征（如有序 / 无序）会直接影响算法执行步骤的多少。

算法时间复杂度的核心思想

分析算法的运行时间需独立于具体机器，排除硬件性能的干扰，聚焦算法本身的执行逻辑。当统一机器性能后，算法的运行时间主要依赖于问题的输入规模和具体的输入实例。

算法的时间复杂度是指算法运行所需的时间，设抽象机上运行的算法 A，某次运行的输入数据规模为 n，输入实例为 I，则算法 A 的运行时间 T 是 n 和 I 的函数，记做 $T(n,I)$。

时间复杂度的分类

根据输入实例的不同分布，算法的时间复杂度分为三种类型，实际分析中常用最坏情况来评估算法，因其能确定算法运行时间的上界，更具一般性：

  1. 最好时间复杂度

    $B(n)=min left{T(n, I) | I in D_{n}right}=Tleft(n, I’right)$

    指在最优的输入实例下，算法运行的最少时间，该情况不常出现，不具备普遍性。

  2. 最坏时间复杂度

    $W(n)=max left{T(n, I) | I in D_{n}right}=Tleft(n, I^{*}right)$

    指在最差的输入实例下，算法运行的最多时间，能确定算法运行时间的上界，是算法分析的主要参考。

  3. 平均时间复杂度

    $A(n)=sum_{I in D_{n}} P(I) T(n, I)$

    指在所有可能的输入实例下，算法运行时间的加权平均值，其中$P(I)$为输入实例 I 出现的概率，该情况分析过程复杂，难度较高。

程序步分析方法

程序步是分析算法时间复杂度的基础方法，分为事前分析和事后测试两类，核心采用事前分析对算法效率进行预判。

基本定义

程序步：在语法或语义上有意义的程序段，其执行时间与问题实例的规模无关，是衡量算法执行步骤的基本单位。

分析示例

  1. 迭代程序（数组累加）

    对于求数组元素累加之和的迭代程序，通过统计各语句的程序步数量，可得时间复杂度为 $T(n)=2n+3$。

  2. 递归程序（数组累加）

    对于递归实现的数组累加程序，设程序步为$T(n)$，可得递推关系：

    $T(n)= begin{cases}2 & n=0 T(n-1)+2 & n>0end{cases}$

    求解得 $T(n)=2n+2$。

渐进复杂性分析

核心思想

渐进分析是算法复杂度分析的核心方法，忽略函数中不显著改变数量级的部分，只关注当问题规模$n to infty$时，算法基本语句执行次数的增长趋势，得到原函数的近似值，该近似值在 n 充分大时会足够接近原值。

渐进时间复杂度：当问题规模充分大时，算法中基本语句的执行次数在渐近意义下的阶，是算法时间复杂度的核心表示形式。

函数增长的规律

当问题规模 n 足够大时，多项式函数中最高次项会主导函数的增长，低次项、常数项、对数项的影响可忽略。例如$f(n)=n^2+100n+log_{10}n+1000$，当 n=10000 时，$n^2$项占比达 99.0%，其余项占比可忽略。

渐近分析的记号

以下分析中，对所有$n$，满足$f(n) ≥0$，$g(n) ≥0$，核心包含 5 种渐近记号，分别表示不同的增长关系：

1. 渐近上界记号 $O$

$O(g(n))={f(n) | 存在正常数c和n_0，使得对所有n ≥n_0，有0 leq f(n) leq cg(n)}$

  • 含义：$f(n)$的增长阶不高于$g(n)$的增长阶，即$f(n)$的增长速度不会快于$g(n)$。

  • 关键：只需找到一组满足条件的$c$和$n_0$即可，前有限个 n 值可不满足不等式；常函数的渐近上界为$O(1)$。

  • 示例：$2n+3=O(n)$（取$c=3，n_0=3$）；$10n^2+4n+2=O(n^2)$（取$c=11，n_0=5$）。

2. 渐近下界记号 $Omega$

$Omega(g(n))={f(n) | 存在正常数c和n_0，使得对所有n ≥n_0，有0 leq cg(n) leq f(n)}$

  • 含义：$f(n)$的增长阶不低于$g(n)$的增长阶，即$f(n)$的增长速度不会慢于$g(n)$。

  • 示例：$2n+3=Omega(n)$（取$c=2，n_0=1$）；$10n^2+4n+2=Omega(n^2)$（取$c=10，n_0=1$）。

3. 非紧上界记号 $o$

$o(g(n))={f(n) | 对于任何正常数c>0，存在n_0>0，使得对所有n ≥n_0，有0 leq f(n)<c g(n)}$

  • 等价形式：$lim_{n to infty} frac{f(n)}{g(n)} = 0$。

  • 含义：$f(n)$的增长阶严格低于$g(n)$的增长阶。

4. 非紧下界记号 $omega$

$omega(g(n))={f(n) | 对于任何正常数c>0，存在n_0>0，使得对所有n ≥n_0，有0 leq c g(n)<f(n)}$

  • 等价形式：$lim_{n to infty} frac{f(n)}{g(n)} = infty$。

  • 含义：$f(n)$的增长阶严格高于$g(n)$的增长阶。

5. 紧渐近界记号 $Theta$

$Theta(g(n))={f(n) | 存在正常数c_1、c_2和n_0，使得对所有n ≥n_0，有c_1g(n) leq f(n) leq c_2g(n)}$

  • 核心定理：$Theta(g(n))=O(g(n)) cap Omega(g(n))$。

  • 含义：$f(n)$的增长阶等于$g(n)$的增长阶，是最精准的渐近复杂度表示。

  • 示例：$n^2+n=Theta(100n^2)$，表示二者增长趋势一致。

渐近记号的关系类比

为便于理解，渐近记号的增长关系可类比为数值大小关系：

  • $f(n)=O(g(n)) approx a leq b$

  • $f(n)=Omega(g(n)) approx a geq b$

  • $f(n)=Theta(g(n)) approx a=b$

  • $f(n)=o(g(n)) approx a<b$

  • $f(n)=omega(g(n)) approx a>b$

多项式的渐近复杂度定理

若$f(n)=a_m n^m+a_{m-1}n^{m-1}+…+a_1n+a_0$是 m 次多项式，且$a_m>0$，则：

  • $f(n)=O(n^m)$

  • $f(n)=Omega(n^m)$

  • $f(n)=Theta(n^m)$

即 m 次多项式的渐近复杂度由最高次项$n^m$决定。

算法的时间复杂度分类

根据渐近时间复杂度的增长趋势，算法分为多项式时间算法和指数时间算法两类，指数时间算法仅在 n 很小时具有实际价值。

常见复杂度的增长顺序

多项式时间算法（增长速度由慢到快）：

$O(1)<O(log n)<O(n)<O(n log n)<O(n^2)<O(n^3)$

指数时间算法（增长速度由慢到快）：

$O(2^n)<O(n!)<O(n^n)$

常用渐进函数增长特征

当输入规模 n 增大时，指数时间算法的执行次数会呈爆炸式增长，而多项式时间算法增长相对平缓，如下表（函数值随 n 的变化）：

递推方程

基本定义

递推方程（也称为递推关系）是自然数上的一个函数$T(n)$，它使用一个或多个小于 n 时的函数值的等式或不等式来描述。

  • 边界条件：递推方程必须具备的初始条件，是求解递推方程的基础（如$T(1)=1$）。

  • 应用场景：递归算法的时间复杂度分析常通过建立递推方程求解。

递推方程的求解方法

核心有三种求解方法：替换法、迭代法、主方法，其中主方法适用于特定形式的递推方程，是递归算法分析的常用方法。

1. 替换法

核心步骤：

  1. 根据递推方程的小值实例（如 n=1,2,3）猜测解的形式；

  2. 用数学归纳法证明猜测的解满足递推方程。

示例（汉诺塔问题）：

汉诺塔问题的递推方程为：

$T(n)= begin{cases}1 & n=1 2 T(n-1)+1 & n>1end{cases}$

  • 猜测：由$T(3)=7=2^3-1$、$T(4)=15=2^4-1$，猜测$T(n)=2^n-1$；

  • 证明：

    • 基例：n=1 时，$T(1)=2^1-1=1$，成立；

    • 归纳假设：假设 k<n 时，$T(k)=2^k-1$；

    • 归纳步骤：$T(n)=2T(n-1)+1=2(2^{n-1}-1)+1=2^n-1$，成立。

  • 结论：$T(n)=Theta(2^n)$。

2. 迭代法

核心步骤：

  1. 不断扩展递推方程，将其转化为一个和式；

  2. 计算和式的结果，得到算法的渐近复杂度。

示例（汉诺塔问题）：

$begin{aligned} T(n) & =2 T(n-1)+1 & =2(2 T(n-2)+1)+1=2^2 T(n-2)+2+1 & =2^3 T(n-3)+2^2+2+1 & vdots & =2^{n-1} T(1)+sum_{i=0}^{n-2}2^i =2^{n-1}×1 + (2^{n-1}-1) =2^n-1 end{aligned}$

3. 主方法

适用场景：求解形如$T(n)=aT(n/b)+f(n)$的递推方程，其中$a ≥1$、$b>1$为常数，$f(n)$为渐近正函数，$n/b$指$lfloor n/b rfloor$或$lceil n/b rceil$。

  • 参数含义：

    • $a$：归约后的子问题个数；

    • $n/b$：归约后每个子问题的规模；

    • $f(n)$：归约过程及组合子问题解的工作量。

主定理：

设$a ≥1$，$b>1$为常数，$f(n)$为渐近正函数，$T(n)=aT(n/b)+f(n)$，则：

  1. 若$f(n)=O(n^{log_b a-varepsilon})$（$varepsilon>0$），则$T(n)=Theta(n^{log_b a})$；

    （$f(n)$增长慢于$n^{log_b a}$，子问题求解主导复杂度）

  2. 若$f(n)=Theta(n^{log_b a})$，则$T(n)=Theta(n^{log_b a} log n)$；

    （$f(n)$与$n^{log_b a}$增长同阶，子问题 + 组合过程共同主导）

  3. 若$f(n)=Omega(n^{log_b a+varepsilon})$（$varepsilon>0$），且存在常数$c<1$，对所有充分大的 n 有$a f(n/b) ≤c f(n)$，则$T(n)=Theta(f(n))$；

    （$f(n)$增长快于$n^{log_b a}$，组合过程主导复杂度）

主定理应用示例：

  1. $T(n)=16T(n/4)+n$：$a=16,b=4,log_416=2$，$f(n)=n=O(n^{2-1})$，符合情况 1，$T(n)=Theta(n^2)$；

  2. $T(n)=T(3n/7)+1$：$a=1,b=7/3,log_{7/3}1=0$，$f(n)=1=Theta(n^0)$，符合情况 2，$T(n)=Theta(log n)$；

  3. $T(n)=3T(n/4)+nlogn$：$a=3,b=4,log_43≈0.793$，$f(n)=nlogn=Omega(n^{0.793+varepsilon})$，且$3×(n/4)log(n/4) ≤ (3/4)nlogn$，符合情况 3，$T(n)=Theta(nlogn)$。

递归树

递归树是迭代法的可视化形式，可形象地展示递推式的迭代过程，通过逐层计算每一层的工作量，求和得到总复杂度。

  • 示例 1：$T(n)=2T(n/2)+n$，递归树每层工作量为 n，共$logn+1$层，总复杂度$T(n)=Theta(nlogn)$；

  • 示例 2：$T(n)=T(n/3)+T(2n/3)+n$，递归树每层工作量为 n，树高$log_{3/2}n$，总复杂度$T(n)=Theta(nlogn)$。

经典算法的复杂度分析示例

1. 插入排序

插入排序的最坏时间复杂度递推方程：

$W(n)= begin{cases}0 & n=1 W(n-1)+n-1 & n>1end{cases}$

求解得：$W(n)=frac{n(n-1)}{2}=Theta(n^2)$，最好时间复杂度为$O(n)$（输入数组已有序）。

2. 二分归并排序

二分归并排序的时间复杂度递推方程：

$W(n)= begin{cases}0 & n=1 2W(n/2)+n-1 & n>1end{cases}$

由主定理，$a=2,b=2,log_22=1$，$f(n)=n-1=Theta(n^1)$，符合情况 2，得$W(n)=Theta(nlogn)$。

本章核心小结

本章围绕算法分析基础展开，核心知识点可概括为三个方面：

  1. 算法复杂度：明确时间复杂度的定义、分类（最好 / 最坏 / 平均），掌握程序步分析方法；

  2. 渐进表示法：理解 5 种渐近记号（$O、Omega、o、omega、Theta$）的定义和含义，掌握多项式时间算法与指数时间算法的增长规律；

  3. 递推关系：掌握递推方程的定义，熟练运用替换法、迭代法、主方法求解递推方程，其中主方法是递归算法复杂度分析的核心工具。

算法分析的核心目标是排除硬件干扰，聚焦算法本身，通过渐近复杂度判断算法的效率，为算法的设计和改进提供理论依据。

定理2-3

如果 ( f(n) = a_m n^m + a_{m-1} n^{m-1} + cdots + a_1 n + a_0 ) 是 ( m ) 次多项式，且 ( a_m > 0 )，则 ( f(n) = Omega(n^m) )。

证明

令 ( K = |a_{m-1}| + |a_{m-2}| + cdots + |a_0| )。
所以 ( a_{m-1}n^{m-1} + a_{m-2}n^{m-2} + cdots + a_0 le |a_{m-1} + a_{m-2} + cdots + a_0| n^{m-1} le K n^{m-1} )。
因此 ( f(n) = a_m n^m + a_{m-1}n^{m-1} + a_{m-2}n^{m-2} + cdots + a_0 > a_m n^m - K n^{m-1} )。

此时 ( n ge frac{2K}{a_m} )。

对所有 ( n ge n_0 )，有 ( f(n) ge frac{a_m}{2} n^m )。

题目描述

表达式一般由操作数(Operand)、运算符(Operator)组成，例如算术表达式中，通常把运算符放在两个操作数的中间，这称为中缀表达式(Infix expression)，如A+B。
波兰数学家Jan Lukasiewicz提出了另一种数学表示法，它有两种表示形式：把运算符写在操作数之前，称为波兰表达式(Polish expression)或前缀表达式(Prefix expression)，如+AB；把运算符写在操作数之后，称为逆波兰表达式(Reverse Polish expression)或后缀表达式(Suffix expression)，如AB+；前后缀表达式的出现是为了方便计算机处理。
本题求解波兰表达式的值，其中运算符包括+ - * /四个。

输入

输入为一行，其中运算符和运算数之间都用空格分隔，运算数是浮点数。
