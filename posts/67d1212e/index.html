操作系统概述

操作系统的定义和地位

定义: 一组控制和管理计算机硬件和软件资源、合理地对各类作业进行调度以及方便用户的程序的集合。

硬件系统 (裸机):CPU、存储器（主存、辅存）、I/O、I/O 控制系统。

软件系统：系统软件、应用软件。

系统软件：管理计算机本身的操作。如：操作系统、编译系统。

应用软件：提供给用户进行解题。如：科学计算、事务管理

操作系统的目标

执行用户程序并使用户问题更易解决

  • 有效性：以一种有效的方式使用系统资源。

  • 方便性：使计算机系统更易于使用。

  • 可扩充性：采用新的 OS 结构，易于功能的增、删、改。

  • 开放性：要求提供统一开放的环境，各种类型的计算机硬件系统，出自不同的厂家，能通过网络集成并能够正确、有效地协同工作，实现应用程序的移植。

操作系统的作用

  1. 作为用户与计算机硬件系统之间的接口

  • 命令方式 (UNIX、DOS 命令)

  • 系统调用方式 (API)

  • 图形、窗口方式 (Windows)

  1. OS 作为计算机系统资源的管理者

  • 硬件资源：处理机（分配控制）、存储器（分配回收）、I/O 设备（分配回收 + 操纵）

  • 信息资源：程序和数据（文件的存取、共享、保护）

  1. OS 实现了对计算机资源的抽象
  • 覆盖 OS 后形成虚拟计算机，用户无需了解物理接口实现细节，简化 I/O 操作

操作系统的发展过程

推动操作系统发展的主要动力

  • 不断改善计算机效率

  • 方便用户，用户不断提出越来越多新的苛刻要求

  • 硬件不断更新换代，数量不断增加，成本不断下降

  • 计算机体系结构的不断发展，计算机功能和复杂性不断增加

  • 不断提出新的应用需求

无操作系统的计算机系统

  1. 人工操作方式

  • 对应第一代计算机（电子管）：体积大、成本高、无外存、机器 / 汇编语言，仅科学计算

  • 特点：单操作员单终端、单任务、独占全机、CPU 等待人工操作、串行执行

  • 缺点：资源浪费、CPU 等待人工操作

  • 核心矛盾：人机矛盾（机器等人）、CPU 与 IO 速度不匹配矛盾（CPU 等 IO）

  1. 解决方法

  • 通道技术、缓冲技术

  • 监督程序

  • 脱机输入输出方式：引入外围机，将程序 / 数据先录入磁带，CPU 高速读取；结果先存磁带再由外围机输出，减少 CPU 空闲、提高 I/O 速度

单道批处理系统

  1. 对应第二代计算机（晶体管）：速度提升、有外存（磁带 / 磁盘）、高级语言（FORTRAN/COBOL）、区分设计与操作

  2. 核心思想：操作员将一批作业脱机输入磁带，由监督程序控制作业逐个自动处理

  3. 典型实例：IBM FMS、IBSYS，密歇根大学 UMES

  4. 特征：自动性、顺序性、单道性

  5. 优点：减少人工操作时间，解决人机矛盾，提高资源利用率和系统效率

  6. 缺点：资源利用率仍低（CPU 等待 IO）、程序调试除错困难

多道批处理系统

  1. 对应第三代计算机（半导体集成电路）：磁盘外存、百万次运算、C/Pascal 语言、大型机为主流（IBM 360）

  2. 核心技术 - 多道程序设计：内存同时驻留多个作业，宏观同时运行、微观交替执行

  3. 系统工作方式：作业存外存形成后备队列，OS 按算法选作业调入内存，运行完毕后交付结果，典型为 IBM OS/360

  4. 特征：多道性、无序性、调度性（作业调度 + 进程调度）

  5. 优点：资源利用率高、系统吞吐量大，标志操作系统正式形成

  6. 缺点：作业平均周转时间长、交互性差，用户无法管理程序，开发调试困难

分时系统

  1. 产生背景：用户对人机交互、共享主机、方便上机的需求，批处理系统无法满足

  2. 定义：一台主机连接多台终端，允许多用户共享资源，用户通过终端交互使用计算机

  3. 核心思想：采用时间片轮转技术，将 CPU 划分为时间片，轮流为终端用户服务，保证快速响应

  4. 实现关键：及时接收（多终端卡、多路输入缓冲区）、及时处理（交互作业驻内存、限制用户数、缩短时间片）

  5. 四大特征：多路性、独立性、及时性（响应 1~3 秒）、交互性

  6. 适用场景：办公自动化、教学、事务处理等需要人机会话的场景

实时系统

  1. 引入原因：实时控制（工业生产、军事武器、飞机控制）、实时信息处理（银行、订票、情报检索）对响应时间要求更高

  2. 定义：所有任务均在规定时间内完成，满足时序可预测性的操作系统

  3. 分类

  • 按任务执行：周期性、非周期性

  • 按截止时间要求：硬实时任务（严格遵守）、软实时任务（不严格）

  1. 设计目标：严格的时间响应、高可靠性、安全性、完整性

  2. 与分时系统对比：多路性、独立性相同；实时系统及时性、可靠性更高，分时系统交互性更强

后续发展的操作系统类型

  1. 嵌入式操作系统

  • 定义：用于嵌入式系统的 OS，为特定功能设计

  • 典型实例：µC/OS-II、嵌入式 Linux、Windows Embedded、VxWorks、Android、iOS

  • 特点：内核小、系统精简、可配置、高实时性

  1. 网络操作系统

  • 定义：网络环境下管理控制网络资源，实现数据通信和资源共享的软件集合

  • 典型实例：UNIX、Linux、Windows NT/2000/Server

  • 核心功能：数据通信、应用互操作、网络管理

  1. 分布式操作系统

  • 分布式系统：多个处理机通过通信线路互连的松耦合系统，特征为分布性、透明性、同一性、全局性

  • 定义：配置在分布式系统上的公用 OS

  • 典型实例：万维网、鸿蒙 OS

  • 核心功能：单处理机 OS 功能 + 网络 OS 功能 + 通信管理、资源管理、进程管理

操作系统的基本特征

并发（最核心特征，其他特征以其为前提）

  1. 并行 vs 并发：并行是同一时刻多个事件发生；并发是同一时间间隔内多个事件发生，单处理机中宏观同时、微观交替

  2. 核心载体：进程（资源分配基本单位）、线程（独立调度基本单位，提高并行度）

  3. 影响：有效提高资源利用率和系统吞吐量，但使系统复杂化，OS 需控制管理并发活动（切换、隔离、协作、资源竞争、数据一致性）

共享

  1. 定义：系统资源可供内存中多个并发进程共同使用

  2. 共享方式

  • 互斥共享：一段时间仅允许一个进程访问，此类资源为临界资源

  • 同时访问：宏观同时，微观交替访问

  1. 核心问题：需解决资源分配、信息保护、存取控制等问题

  2. 与并发的关系：互为存在条件，并发是共享的前提，共享是并发的保障

虚拟

  1. 定义：通过技术将一个物理实体变为若干个逻辑对应物

  2. 作用：OS 管理资源的重要手段，提高资源利用率

  3. 实现技术

  • 时分复用：虚拟处理机、虚拟设备

  • 空分复用：虚拟磁盘、虚拟存储器

异步

  1. 定义：多道程序环境下，进程执行 “走走停停”，执行时间、顺序、推进速度不可预知

  2. 原因：进程竞争资源、硬件 / 软件中断随机、用户命令随机、程序错误随机

  3. 特点：进程以不可预知的速度向前推进，OS 需保证异步执行的进程最终都能完成

操作系统的主要功能

处理机管理

  1. 主要任务：分配处理机，对其运行进行有效控制和管理，核心是进程管理

  2. 核心功能

  • 进程控制：创建、撤销、状态转换

  • 进程同步：信号量机制

  • 进程通信：直接通信、间接通信

  • 调度：作业调度（分配资源、建立进程、进入就绪队列）、进程调度（选就绪进程分配 CPU），调度算法如 FCFS、优先权

存储器管理

  1. 主要任务：为多道程序提供运行环境，方便用户使用，提高利用率，逻辑扩充内存

  2. 核心功能

  • 内存分配和回收

  • 内存保护：保证程序仅在自身内存空间运行，不访问 OS 程序和数据

  • 地址映射：将逻辑地址转换为物理地址

  • 内存扩充：虚拟存储技术（请求调入、置换功能）

设备管理

  1. 主要任务：完成用户 I/O 请求，分配 I/O 设备，提高设备利用率和速度，方便用户使用

  2. 核心功能：缓冲管理、设备分配、设备处理、虚拟设备

文件管理

  1. 主要任务：管理用户和系统文件，方便使用，保证文件安全性

  2. 核心功能：文件存储空间管理、目录管理、文件的读写管理和保护

操作系统与用户之间的接口

  1. 用户接口

  • 联机用户接口：命令行方式（CLI）

  • 脱机用户接口：作业说明书（作业控制语言 JCL）

  • 图形用户接口（GUI）

  1. 程序接口：系统调用（完成特定功能的子程序）

现代 OS 的新功能

  1. 系统安全：认证、密码、访问控制、反病毒技术

  2. 网络功能和服务：网络通信、资源管理、应用互操作

  3. 支持多媒体：接纳控制、实时调度、多媒体文件存储

操作系统的结构设计

设计原则

实现基本管理功能和用户接口；保证性能、保护和安全、可靠性（正确性、稳定性）；提升可维护性、可理解性；兼顾商业化因素、标准和开放系统（集成、移植、互操作）

设计特征

复杂程度高、生成周期长、正确性难保证（如 CTSS、OS/360、Multics、Windows 2000）

主要结构类型

  1. 无结构操作系统

  • 特点：早期设计，仅关注功能实现和效率，无统一设计思想，编制紧凑程序，无限制使用 goto

  • 缺点：逻辑复杂，难以维护和理解

  1. 模块化操作系统

  • 特点：将 OS 分解为多个模块（如进程、内存模块），模块内再分子模块，通过分解控制复杂度

  • 优点：提高可维护性和适应性，并行开发加速进程

  • 缺点：接口难以确定，大型软件中模块依赖关系复杂

  1. 分层式操作系统

  • 特点：将 OS 划分为多个层次，下层为上层提供服务，单向依赖；低层为公用模块，高层为用户接口

  • 层次划分（从低到高）：硬件层→核心层→设备驱动层→系统调用接口层→服务层→应用层

  • 优点：简化设计复杂度，便于调试和维护

  • 缺点：系统效率降低，功能执行需穿越多层，增加通信开销

  1. 微内核 OS 结构

  • 基本概念：足够小的内核；遵循 “机制与策略分离”；基于客户 / 服务器模式；采用面向对象技术

  • 内核基本功能：进程管理、低级存储器管理、中断和陷入处理

  • 典型实例：Mach OS、Windows 2000/XP

  • 优点：可扩展性高、可靠性强、可移植性好、支持分布式系统、融入面向对象技术

  • 缺点：运行效率降低，客户请求需多次消息交互和用户 / 内核模式切换

  • 核心思想：将内存管理、设备管理、文件系统等从内核分离为独立模块，内核仅保留最基本功能，简洁可靠

研究操作系统的几种观点

从不同角度理解 OS 的本质，核心围绕资源管理、用户接口、虚拟机器等核心定位，是对前述 OS 定义、作用、特征的综合解读和延伸。

流行的操作系统

涵盖前述分时、网络、嵌入式、分布式等类型中的典型系统，如 UNIX、Linux、Windows 系列、Android、iOS、鸿蒙 OS 等，各系统依托不同 OS 结构和技术，适配不同应用场景（桌面、服务器、移动、嵌入式、分布式）。
